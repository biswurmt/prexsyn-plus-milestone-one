\section*{Data and Preprocessing}

We sample 100 molecules from ChEMBL with pre-stratified difficulty. PrexSyn is first run on a larger pool to compute baseline quality scores via ECFP4 Tanimoto similarity (a 0-to-1 measure of shared circular substructure features between two molecular fingerprints) between the best candidate and the specification. Approximately 25 molecules are then drawn from each of four quality bins ($<$0.5, 0.5--0.7, 0.7--0.85, 0.85--1.0), ensuring balanced representation across difficulty levels.

For each specification, PrexSyn generates 256 candidates; the highest-scoring candidate ($K=1$) seeds all five modification methods, so results reflect the modification method rather than seed diversity. Top-$K$ seeding ($K>1$) is a natural extension that tests whether seed diversity compounds with method diversity; we fix $K=1$ here for clean attribution. Each method produces up to $N$ variants; $N$ is set by a pilot saturation study (5 specs, $N=1{,}000$ per method) to identify the diversity plateau beyond which additional variants yield diminishing structural novelty. Expected cap: 100--500. Variants are deduplicated by canonical SMILES (a standardized text representation of molecular structure) before synthesizability checking via AiZynthFinder at search depth 1, with depth-2 sensitivity analysis on a 10\% subsample to quantify routes missed by shallower search. Estimated total load after deduplication: $\sim$100K unique molecules, $\sim$10 core-hours.

No models are trained in this study; all methods are used as pre-trained or pre-configured tools. CReM's fragment database derives from ChEMBL, the same source as our specifications. Since ChEMBL molecules serve only as property-profile sources, not reconstruction targets, this overlap reflects realistic tool usage rather than information leakage.
