\section*{Proposed methods}

The benchmarking pipeline seeds four modification methods (CReM, mmpdb, JT-VAE, and LibINVENT) with PrexSyn's top-scoring candidate per specification. Each method generates a set of structural variants, which we screen through a shared synthesizability and property-conservation pipeline alongside an equal number of additional PrexSyn draws from the same specification. This repeated-sampling baseline establishes what PrexSyn alone achieves with more compute, so we evaluate each modification method by how many viable candidates it produces that repeated sampling cannot.

\textit{CReM} \cite{bib7} replaces molecular fragments, including side chains, ring systems, and functional groups, with alternatives drawn from its ChEMBL-derived database while enforcing chemical validity. A context radius controls the edit scale: small radii make conservative R-group substitutions; larger radii enable ring and scaffold-level replacements. We benchmark all three radii to characterize the edit-scale/quality tradeoff.
\textit{mmpdb} \cite{bib8} applies matched molecular pair rules, single-site substitutions extracted from pairs of real compounds that differ by exactly one structural change (e.g., methyl $\rightarrow$ chloro, phenyl $\rightarrow$ pyridyl). Empirical property-change annotations guide each transformation toward property conservation, making these the most conservative edits in our benchmark.
\textit{JT-VAE} \cite{bib9} encodes molecules into a continuous latent space by decomposing each structure into a junction tree, a hierarchy of chemically meaningful substructures such as ring systems, bonds, and atom clusters. Perturbing the seed's latent vector and decoding the result enables scaffold hopping, replacing an entire core structure while preserving the pharmacological profile. This global reach exceeds what any fragment-level method can achieve.
\textit{LibINVENT} \cite{bib10,bib11} preserves the molecular scaffold from PrexSyn's output, retaining the core ring system and connectivity, while generating alternative R-group decorations via reinforcement learning steered toward target properties.

We selected these four methods from a systematic survey of $\sim$50 candidates to maximize coverage across two dimensions: rule-based (CReM, mmpdb) versus learned (JT-VAE, LibINVENT) paradigms, and local versus global edit scale. No prior work benchmarks modification methods as post-generation complements to synthesis-aware generation. Because each variant must independently pass retrosynthetic verification, the results directly quantify how far each method can push chemical novelty before synthesizability degrades. Our complementarity analysis further identifies which combinations maximize total unique-hit coverage, informing pipeline design for medicinal chemistry and automated discovery.
